ðŸ¤– FOREIGN KEY ðŸ¤–

ðŸ‘¾ The FOREIGN KEY constraint is used to prevent actions that would destroy links between tables.

ðŸ‘¾ A FOREIGN KEY is a field (or collection of fields) in one table, that refers to the PRIMARY KEY in another table.

ðŸ‘¾ The table with the foreign key is called the child table, and the table with the primary key is called the referenced or parent table.

CREATE TABLE customers (
    customer_id INT,
    customer_name VARCHAR(100) NOT NULL,
    customer_email VARCHAR(100) NOT NULL,
    customer_phone VARCHAR(15) NOT NULL,
    PRIMARY KEY (customer_id)
);

CREATE TABLE orders (
    order_id INT,
    order_date DATE,
    amount DECIMAL(10, 2),
    customer_id INT,
    PRIMARY KEY (order_id),
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

ðŸ‘¾ To create a FOREIGN KEY constraint on the column when the "Orders" table is already created

ðŸ‘€ ALTER TABLE Orders ADD FOREIGN KEY (PersonID) REFERENCES Persons(PersonID);

ðŸ‘¾ To drop a FOREIGN KEY constraint

ðŸ‘€ ALTER TABLE Orders DROP FOREIGN KEY FK_PersonOrder;
